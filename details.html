Relative Path in Unix

Relative Path

A relative path specifies the location of a file or directory in relation to the current directory. It is commonly used when navigating a file system or referencing files in code.

Usage:
$ docs/file.txt

Example:

Current Directory: /home/user/
Target File: /home/user/docs/file.txt
Relative Path: docs/file.txt

Key Components in a Pathname:

  • `.`: Refers to the current directory.
  • `..`: Refers to the parent directory (one level up).
  • `~`: Represents the home directory of the current user.

Example:

Usage:
$ ./script.sh
$ ../docs/file.txt
$ ~/projects/code.py

Steps to Find a Pathname:

  1. Step 1: Open a terminal (e.g., Cygwin, Command Prompt, or Bash).
  2. Step 2: Use the pwd command to display the current working directory. 
    $ pwd
/home/user
  3. Step 3: List the files and directories in the current directory using the ls command. 
    $ ls
docs  script.sh
  4. Step 4: Navigate to the desired directory using cd. 
    $ cd docs
  5. Step 5: Identify the relative or absolute path of the desired file. 
    Relative Path: docs/file.txt
Absolute Path: /home/user/docs/file.txt
