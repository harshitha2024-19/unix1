Topic: Pathnames

Learn about the components of pathnames in Unix and how they are used to locate files and directories.

What is a Pathname?

A pathname refers to the location of a file or directory within the file system. It can be either a relative or absolute path.

Types of Pathnames

  • Absolute Path: An absolute path refers to the complete path from the root directory, starting with a `/`. It is independent of the current working directory.
  • Relative Path: A relative path refers to a path relative to the current working directory. It does not start with `/` and depends on where you are located in the file system.

Key Components in Pathname

  • `.` (dot): Refers to the current directory.
  • `..` (double dot): Refers to the parent directory.
  • `~` (tilde): Refers to the home directory of the current user.

Steps to Find a Pathname

  1. Identify the starting point (current working directory).
  2. Use relative or absolute references to navigate through the file system.
  3. Use the appropriate components (`/`, `.`, `..`, `~`) based on the context.

For example, if your current working directory is `/home/user` and you want to navigate to `/home/user/documents`, you would use the relative path `documents`.
